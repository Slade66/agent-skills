# weekly-git-report

## 作用

根据 Git 提交自动生成本周周报的对外汇报文案（中文、有序、简洁）。

## 步骤

1. 查询本项目的 git 本周（周一到周五）的提交信息
   - 如果commit message 太模糊，你要去查看提交了什么文件，然后看看实现了什么功能（拆成原子的功能）
2. 将提交结果压缩为原子任务（保留顺序）。
3. 输出汇报文本。

## 输出要求

1. 中文序号格式：`一、二、三…`
2. 每条一句话，聚焦“完成了什么，解决了什么”
3. 不编造、不过度展开技术细节

## 输出模板

```text
本周完成的工作：
一、搭建迁移任务工作空间隔离机制，避免多任务并发时的资源互扰与状态污染。
二、补充项目背景与需求说明文档，统一需求口径并完善交付前说明材料。
二、整理并形成 Bifrost 在线迁移原理汇报大纲，明确迁移架构与关键流程。
四、修复 Linux 驱动注入日志路径，保障迁移日志可定位、可追踪、可排障。
五、修正目标虚拟机启动固件映射配置，消除迁移后因固件参数不匹配导致的启动异常。
六、优化 FC 系统模板映射逻辑，提升操作系统与模板匹配的正确性与兼容覆盖。
七、新增 FusionCompute OS 版本选择能力，实现按目标系统版本的细粒度迁移配置。
八、实现 cutover 就绪状态的 WebSocket 推送，支持前置割接动作的实时可观测。
九、重构任务 OS schema 迁移流程为 SQLite 安全实现，降低数据库并发和一致性风险。
十、补充 Kylin/Euler 驱动注入实施方案文档，明确适用边界、处理规则与风险说明。
十一、增强 Kylin/Euler 驱动注入兼容性与稳定性，覆盖关键注入场景并修复相关缺陷。
十二、调整驱动注入策略为“跳过内置 virtio驱动注入”，避免重复注入引发的冲突与迁移失败。
```
